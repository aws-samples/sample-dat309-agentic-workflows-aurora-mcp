From a search box to a shopping partner.

Chapter 01

Five pillars for the 2026 narrative.

Each pillar maps to an industry pattern that matured in 2025–26 — durable memory, multi-agent supervision, declarative tool calls via MCP, governed autonomous actions, and observability as a first-class product surface. Together they shape the story Meridian should tell.

Pillar 01

Durable memory, not just sessions.

Pgvector + a dedicated profile schema in Aurora hold long-lived facts: sizes, brand affinities, prior orders, allergies, return reasons. Memory is read on every turn, written by an explicit writer agent, and presented to the user so they can edit or forget.

Why it matters. 2026 users expect agents to remember "I run in 11s, hate ankle support, returned the last Hokas". Without it the demo plateaus at single-turn search.
Pillar 02

From phases to a supervisor.

Replace the linear Phase 1/2/3 toggle with a supervisor that fans out to specialist agents — Search, Inventory, Policy, Payment, CX. Each owns a narrow MCP toolset and is independently swappable. Phases become "modes" the supervisor picks per query.

Why it matters. The phase pills today demo a teaching ladder. A supervisor graph demos a system you'd actually run.
Pillar 03

MCP as the contract with everything.

Every tool — Aurora reads, Stripe charges, ShipStation labels, Zendesk tickets, internal pricing rules — lives behind an MCP server with versioned schemas. The UI surfaces a "Tool Catalog" page where you can inspect, version-pin, and dry-run each one.

Why it matters. MCP went from novel to assumed. Showing a versioned, inspectable catalog is the modern equivalent of an API explorer.
Pillar 04

Governed autonomy.

Agents act under a budget: dollars per session, ms per call, tools per plan, and confirmation thresholds. Anything risky — an actual charge, a policy override, a refund above a limit — pauses for human approval inline in the chat with a diff of what will happen.

Why it matters. 2026 procurement won't approve agents that can't be reined in. Show the leash; show that it works.
Pillar 05

Observability as a surface.

The current Activity Panel becomes a full Trace view: a Gantt-style timeline of agents, tool calls, SQL, retrieval, latencies, token spend, and cache hits. Every trace is replayable and shareable as a permalinked URL.

Why it matters. The thing developers want to copy is not the chat — it's the trace. Make it the centerpiece.
Pillar 06

The graph under the catalog.

Aurora's relational core stays — but layer Apache AGE for product-relationship queries: "shoes that customers who liked the Pegasus also bought, filtered by what's in stock in my size". Hybrid: vector + lexical + graph traversal, in one SQL query.

Why it matters. Graph + vector + relational in one engine is Aurora's best 2026 story. Make it a visible, demo-able feature.

Surface 06 · Governed action

Plan, confirm, commit.

Before any irreversible action — placing the order, refunding, applying a discount — the agent presents a plan with a diff. The user confirms once. The receipt links to the trace.
